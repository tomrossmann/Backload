Features

Backload has built-in support for complex storage structures without a single line of code, automatic thumbnail creation, image processing, support for several client side file upload plugins, file bundeling, security, and more. When it comes to exceptions or upload problems, the tracing feature helps to identify errors. All features can be customized declaratively (config file) or in your code. With release 2.0 Backload also supports uploads to a cloud storage.

Highly configurable

Backload is highly configurable - declaratively (in the Web.config or an external config file) or in code. Some settings are storage folder structure or storage locations, image processing options, content-type file handling, security features, url patterns and more.

Image processing

The image processing feature enables you to generate thumbnails for uploaded files or resize images before they are stored. Options are width and height of the resulting image, resize modes, new target image type, dpi, canvas color or thumbnail path.
 

Content-type handling

An uploaded file can be processed based on the its file type. For example, when you allow users to upload files like PDF files, Word files or images you may want to return related thumbnails. This feature also allows you to store files in a content type folder (e.g. /pdfs/ subfolder for PDF files).

Cloud storage

Starting with release 2.0 Backload will support cloud storage, enabling you to upload, retrieve or delete files in the cloud with the major cloud providers.

Security

Backload has security features included already. Roles based authorization enables you to restrict user access on uploads, downloads or file deletion.

Tracing

The Backload component supports tracing with common .NET trace listeners. You can log trace messages to log files, databases or logging frameworks (e.g. log4net).

Scalability

Backload is used by customers with 100000 and more uploads per day! Internally it uses various none-blockable asynchronous programming patterns.

Zero configuration

Backload has many configuration options and this may lead to confusion. If you are happy with the defaults, set only those options you want to change.
